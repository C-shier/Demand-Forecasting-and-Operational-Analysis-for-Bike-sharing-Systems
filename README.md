# 共享单车需求预测与运营分析

## 项目来源

数据来源为 UCI Machine Learning Repository 的 Bike Sharing Dataset，原始分析对象为 `hour.csv` 小时级租赁明细。本发布版保留分析脚本、Notebook、SQL、聚合结果表和图表，不重复上传原始数据集文件。

| 项目项 | 内容 |
|---|---|
| 数据主题 | 共享单车小时级租赁需求 |
| 时间范围 | 2011-01-01 至 2012-12-31 |
| 记录数 | 17,379 |
| 总租赁量 | 3,292,679 |
| 小时平均租赁量 | 189.46 |

## 项目背景

共享单车平台需要提前判断不同时间段、不同天气、不同季节下的租赁需求，以便安排车辆调度、补车节奏和运营资源。本项目先完成需求规律拆解，再构建小时级租赁量预测模型，形成“能解释业务规律，也能支持预测”的项目展示方案。

## 分析思路

1. 读取小时级租赁数据，完成日期拆分、季节与天气类别映射。
2. 从月度趋势、24 小时分布、工作日与非工作日差异拆解需求节奏。
3. 结合天气、季节、温度、湿度、风速分析需求变化原因。
4. 构建历史均值基线、线性回归、随机森林三类模型对比预测效果。
5. 输出聚合结果表、图表和项目结论，沉淀成适合 GitHub 浏览的项目展示材料。

## 操作过程

```bash
pip install -r requirements.txt
python src/共享单车需求预测与运营分析.py
```

脚本默认读取 `datasets/02_bike_sharing_dataset/hour.csv`，并将结果输出到 `data/tables/` 与 `images/`。如果原始文件不在默认位置，可以先设置环境变量：

```bash
set BIKE_SHARING_HOUR_CSV=你的hour.csv路径
```

## 主要结论

- 共享单车需求具有明显通勤高峰特征，17-18 点是重点补车时段。
- 天气对需求影响很直接，恶劣天气下平均租赁量只有晴天/少云场景的约三分之一。
- 注册用户贡献了 81.17% 的租赁量，说明稳定通勤用户是平台核心人群。
- 随机森林模型效果最好，说明时间、天气和季节因素与租赁需求之间存在明显非线性关系。
- 运营上可以将天气预报、工作日标签和小时级历史需求组合为调度规则，提前安排高峰补车和低谷回收。
